# Welcome to _my-vcard_ project Storybook

[**Storybook**](https://storybook.js.org/) helps to build UI components in
isolation from the app's business logic, data and context.  
That makes it easy to develop hard-to-reach states and makes development faster
and easier.

It's possible to develop entire UIs without needing to start up a complex dev
stack, force certain data into the database, or navigate around the application.  
UI states can be saved as **stories** to revisit during development, testing,
QA or demos.

**Storybook helps to document components for reuse and automatically visually
test them to prevent bugs.**

Browse all stories from this project now by navigating to them in the sidebar.
View their code in the `stories` directory to learn how they work.
Building UIs with a [**component-driven**](https://componentdriven.org) process starting with atomic components and ending with pages is recomended.

Note: on the top toolbar there are several controls wich make possible to set the test canvas zoom, viewport size, etc.
There is also a "Locale Selector" whith the languages available on the application. This selector will not apply any
translations, its purpose is to test components that have behaviors based on the selected language.
